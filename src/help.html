RemoteBorne Manager - Aide

Guide rapide opérateur

1) Démarrage

copy config\config.example.ini config\config.ini
python src/RemoteBorneManager.py

Le runtime utilise config/config.ini.

2) Connexion SSH

  • Connect : lance/reprend la connexion.
  • Disconnect : déconnexion manuelle (la reconnexion auto est bloquée après action manuelle).
  • La version de l'application est affichée au démarrage dans les logs.

3) GridCodes: Browse / Edit / Print

Édition

  • Bouton Edit ou double-clic fichier.
  • Ctrl+F pour rechercher (surlignage + compteur).
  • Save normalise les fins de ligne en LF (évite les ^M).

Impression PDF

  • Nom proposé automatiquement : <nom_fichier>.pdf.
  • Wrap des longues lignes selon largeur réelle pour éviter le texte tronqué.

4) Energy Manager

Mode P/Q

  • Commande standard : power + reactive-power.

Mode CosPhi

  • Procédure mise à jour : 
    EnergyManagerTestingTool --grid-option "SetpointCosPhi_Pct=..." && EnergyManagerTestingTool -S -s ocpp -a --power ... -m CentralSetpoint

5) Debug Logs

  • Start démarre le tail distant sans popup bloquante.
  • Sous Windows, plink est lancé sans fenêtre cmd visible.
  • Options: pause live, auto-scroll, filtres grep/ERROR/WARN/INFO.

6) Network Config

  • Validation host/IP, port (1..65535), paths.
  • Création du dossier local si absent.
  • Sauvegarde recharge la cible SSH et relance la reconnexion.
